class pyfor.collection.CloudDataFrame(*args, **kwargs)¶

Bases: geopandas.geodataframe.GeoDataFrame

Implements a data frame structure for processing and managing multiple Cloud objects. It is recommended to initialize using the from_dir() function.

Initialize self. See help(type(self)) for accurate signature.

bounding_box¶

Retrieves the bounding box for the entire collection. As a tuple (minx, muny, maxx, maxy)

clip(polygons, path, poly_names=None)¶

A collection-level clipping method. This function is meant for efficient querying across the study area using a set of polygons. This method requires the presence of .lax files in the collection directory. To generate these .lax files please use create_index() first. Each polygon will be clipped and written to the specified path.

Parameters:
  • polygons – Either a list or geopandas.GeoSeries of shapely polygons.
  • path – The output path of the clip.
  • poly_names – A list of polygon names to use when writing to file.
create_index()¶

For each file in the collection, creates .lax files for spatial indexing using the default values.

par_apply(func, column='las_path', buffer_distance=0, *args)¶

Apply a function to each item in column. Allows for parallelization using the n_jobs argument. This is achieved via joblib.Parallel and joblib.delayed().

Parameters:
  • func – The user defined function, must accept as an argument the value of each row of column.
  • column – The column to apply on, will be the first argument to func.
  • buffer_distance – The distance to buffer and aggregate each tile.

retile(out_dir, verbose=False)¶

Retiles the collection into out_dir. This is a simplified retiling function that splits each tile into quadrants and writes these new quadrants to disk. The written files will be the original file name with an index appended. 0 is the bottom left quadrant, 1 is the bottom right, 2 is the top left and 3 is the top right.
